1. What is data?

Data is a collection of facts, figures and statistics related to something or someone. After processing data, it is converted to useful information.

For example, the scores of different players in a cricket match are: 23, 45, 9, 17, 94, 57, 12, 3, 15, 1

2. Define data set.

A collection of related records is called a data set or a file. A file is treated as a single unit. A collection of a college's student records will form a students file or data set.

3. What is the difference between data and information?

Data is a set of raw facts and figures, while information is the processed form of data. Data is used as input to the computer, and the computer, after processing, outputs information.

For example, the scores of different players in a cricket match are: 23, 45, 9, 17, 94, 57, 12, 3, 15, 1. This is data.

If we want something useful like the highest score, we can process the above data to find 94. This (highest score) is information.

4. Define data manipulation and data processing.

The process of manipulating data to achieve the required objectives and results is called data processing.

The process of applying different operations on data is called data manipulation. It may include classifying, sorting, summarizing, filtering and performing calculations.

5. What is data capturing?

The process to store the data in some form is called data capturing. Data is captured before it can be processed. Data is stored on source documents, and entered into a computer using input devices.

6. List different activities in data processing.

The different activities involved in data processing are:
  1. Data capturing.
  2. Data manipulation.
  3. Managing output.

7. Define field.

A set of related characters that represent a unit of data is called a field. Each field is given a unique name known as the field name. For example, Roll number, Name, Address and Marks are some fields for a student.

8. What is a record?

A collection of related fields is called a record. A record is treated as a single unit. In MS-Access, each column in a table represents a field while each row represents records.

9. What is the difference between a record and a field?

Fields are independent entities that represent a unit of data. While a record is a collection of related fields.

In MS-Access tables, the columns are fields. While the rows are different records.

If Roll number, Name, Percentage are the fields for a student, then a record will be {13, "Steve Jobs", 83%}.

10. What is a file?

A collection of related records, as single unit, is called file. Files are stored in a computer's hard disk. For example, a student file may contain hundreds of records of a college's students.

11. List file types from functional point of view.

From functional point of view, we can divide files in two types:
  1. Program files.
  2. Data files.

12. List file types from usage point of view.

From usage point of view, we can divide files in three types:
  1. Master file.
  2. Transaction file.
  3. Backup file.

13. What is a data file?

A file that contains data is called data file. Data file is always created by application programs. Each program stores files in a specific format. The program that created a file is needed to open and use the file. Different types of data are recognized by their extension.

14. What is a master file?

Master files are the core files in a file system. They store data required by file system for use in future. These files are updated periodically on a regular basis. Master files are never empty from the time they are created.

15. What is a transaction file?

A file that is used to store input data before processing is called a transaction file. It may be a temporary file. It is usually used to update data in a master file. The transaction file may exist only until the master file is updated.

16. What is a program file?

A file that contains the program instructions, for the CPU to execute, is called a program file. The program files may have an extension of .com or .exe.

17. Define file organization.

A technique for physically arranging the records of a file on secondary storage devices is called a file organization.

18. How many types of file organization are there?

The types of file organization are:
  1. Sequential files.
  2. Direct (or Random) Access files.
  3. Indexed Sequential files.

19. What is a sequential (access) file? What is an indexed sequential (access) file?

The records in sequential file organization are stored in sequence. The sequence means the records are stored one after the other. The records can be retrieved only in the sequence in which they were stored. Magnetic tape is an example of a storage medium which uses sequential storage.

In indexed sequential file organization, records are stored in ascending or descending order based on a value called key. The index file organization maintains an index in a file.

20. What are direct access files?

Records in a direct file organization are not stored in a particular sequence. A key value of a record is used to determine the location to store the record. Each record is accessed directly without going through the preceding records. This file organization is suitable for storing data on disk.

21. What is a database system? Give examples.

A database system is a collection of data as well as programs required to manage that data, a computerized record keeping system. The main purpose of a database system is to maintain data in a structured manner so it is efficient to retrieve and update data when needed.

22. What are the objectives of database (design process)?

The main objectives of a database system are:
  1. Data Integration: Data is logically centralized even if it may be located at different locations.
  2. Data Integrity: Data is reliable and accurate.
  3. Data Independence: Data and the application programs don't depend on each other.

23. What is meant by data integrity and database integrity?

Data integrity refers to the correctness and consistency of data. Integrity is usually expressed in terms of certain constraints. These are the consistency rules that can be applied to database so that only correct data is entered into the database.

Database integrity means almost the same thing. It is to ensure that the data entered into the database is correct.

24. What is data independence?

The separation of data structure of database from the application program that uses the data is called data independence. In a database, you can easily change the structure of database without modifying the application program.

25. What is the difference between data independence and dependence?

In data dependence, the changes made to data organization/structure affect the application programs that use the data. While in data independence, the application programs are not affected by any changes made to the organization/structure of data.

26. Describe the concept of consistency constraints.

Consistency constraints are the rules that must be followed to enter data in the database. If a data does not fulfill these constraints, it cannot be entered in the database.

27. What problems does data redundancy cause?

Redundant data uses additional storage space and makes it difficult to maintain the accuracy of database when changes are made.

28. List different database models.

The main three different database models are:
  • Hierarchical Database Model.
  • Network Database Model.
  • Relational Database Model.

29. What is a Database Management System (DBMS)? What are its objectives?

A Database Management System (DBMS) is a collection of programs, which are used for sorting and manipulating data in a database. It is an improvement over the traditional file management system. Its main purpose is to provide an easy and efficient way of sorting data and retrieving information.

The objectives of a DBMS are:

  • Shareability.
  • Availability.
  • Evolvability.
  • Database Integrity.

30. What is the difference between a database and a Database Management System (DBMS)?

Database is a collection of logically related data sets or files. A DBMS is a collection of program to create and maintain database.

31. State the use of Query language?

A query language is used to perform different operations on databases. The most popular query language is SQL (Structured Query Language). The commands or statements in SQL are written in a format very similar to English sentences.

32. Why is a report generator used in databases?

A report generator is used to produce reports. It retrieves data from database and displays it in different formats. The report format can be specified in advance such as row heading, column heading and page heading etc.

33. What is DDL?

DDL stands for Data Definition Language. It consists of SQL commands used to define a database, creating tables, indexes and views. Some important commands in DDL are: CREATE TABLE, DROP TABLE, ALTER TABLE and CREATE VIEW.

34. What is a data dictionary?

A data dictionary is used to store data definitions or descriptions of the structure of data used in database. It may also monitor the data that is entered. It ensures that the data is according to the data definition rules.

35. What is meant by access security?

Access security refers to the protection of database from unauthorized access. It may include password and access restrictions. Database management system provides the facility of access security.

36. What is the purpose of backup and recovery?

Backup is used to store a copy of important data in database. The recovery process uses the data in backup if the original data is damaged due to any reason.
