1. Define Referential integrity.

The referential integrity is a constraint on foreign key value. It states that if a foreign key exists in a relation, the foreign key value must match the primary key value of some tuple (row) in its parent table. Otherwise the foreign key value must be null.

2. What is Entity integrity?

Entity Integrity is a constraint on primary key. Entity Integrity ensures two properties for primary keys:
  1. The primary key for a row is unique; it does not match the primary key of any other row in the table.
  2. The primary key is not null, no attribute (columns) of the primary key may be set to null.

3. Differentiate between Entity integrity and Referential integrity.

Entity integrity:
  • The entity integrity is a constraint on primary key.
  • Primary key cannot contain null (in any column, in case of a composite key)
Referential integrity:
  • Referential integrity is a constraint on foreign key.
  • A valid foreign key value must always reference an existing primary key or contain a null value.

4. How is Referential integrity achieved?

The referential integrity can be achieved by connecting two relations by specifying relationships between them. While specifying relationships, we must ensure that the foreign key value must always reference an existing primary key or contain a null value.

5. How Entity integrity is maintained?

Entity integrity is attained be specifying a primary key in a table. Then the DBMS automatically takes care of ensuring entity integrity by not allowing null value and non-unique values in the primary key field.

6. What is Partial dependency?

A type of dependency in which one or more non-key attributes are functionally dependent on a part of primary key is called partial dependency.

7. Define determinant?

A determinant is an attribute whose value enables us to obtain the value of other related attributes. It appears on the left side of a functional dependency. Therefore, in A → B, the determinant is A.

8. What is data redundancy?

Redundancy occurs in a relation when same information is stored in two or more different ways. For example, consider:

Employee(Emp_ID, Name, Address, Age, Date-of-birth)

Here, we don't need one Age, since we can calculate it using Date of birth.

9. What is a Synonym?

Synonym is an error which occurs in tables, when two different names are used for the same attribute in different tables. The name of the attributes must be same if it exists in two or more tables.

10. What is a Homonym?

Homonym is an error which occurs in tables when same name is used for two different attributes. For example, consider:

Customer(Customer_ID, Customer_Name, Address)

Supplier(Supplier_ID, Customer_Name, Address)

11. What are database anomalies?

Database anomalies are problems in relations due to the data redundancy. These problems are also referred to as update anomalies. These problems occur when insertion, deletion and modification operations are performed on the database. Some important data may be lost while performing these operations, if the relation contains anomalies.

12. List different types of database anomalies?

The following are the different types of database anomalies:
  • Insertion anomaly.
  • Deletion anomaly.
  • Modification anomaly.

13. What is Normalization?

The process of producing a simpler and more reliable database is called normalization. It is used to create a suitable set of relations for storing data. It identifies and corrects the problems and complexities of database. It produces new set of relations which are as free of problems as possible.

14. When is a relation in First Normal form?

A relation R is said to be in first normal form (1NF), if and only if all underlying domains contain atomic values only. It means that the relation does not contain any column/attribute whose values can be further divided. For example, Address can be divided into House number, Street number, City, Zip code, State/Province, and Country.

15. Define Second Normal form.

A relation R is said to be in second normal form (2NF), if it is in 1NF and every non-key attribute is fully functionally dependent on all parts of the primary key.

16. What is 3NF?

A relation R is said to be in third normal form or 3NF, if it is in 2NF and no transitive dependency exists. The transitive dependency is an important factor in normalization. A relation is not in 3NF if the value of a non-key attribute can be obtained by knowing the value of another non-key attribute.
