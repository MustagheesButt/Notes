1. What is a key?

A key is an attribute or a set of attributes that uniquely identifies a tuple (a record) in a relation (a table). The keys are defined in tables to access stored data quickly and efficiently. They are also used to create relationships between different tables.

2. Define Primary key.

The attribute or combination of attributes that uniquely identifies a row (a record) in a relation is known as primary key. A relation can have only one primary key, and the primary key cannot be null (empty).

3. Define Foreign key.

A foreign key is an attribute or combination of attributes in a relation whose value matches a primary key in another relation. The table in which foreign key is created is known as dependent table. The table to which the foreign key refers is known as the parent table.

4. What is Candidate key?

A candidate key is an attribute, or a combination of attributes, in a relation which can be used to uniquely identify rows in that relation. Candidate keys can be used as primary key.

5. Define Alternate key.

The candidate keys which are not used as the primary key in a relation are called alternate keys.

6. What is Composite key?

A candidate key that consists of two or more attributes is called a composite key. For example, a table in which we store the courses each student is attending, at a university, will have a composite key containing Student ID and Course ID as the composite key.

7. List different types of keys.

Some different types of keys used in a database are:
  • Primary Key.
  • Foreign Key.
  • Secondary Key.
  • Candidate Key.
  • Alternate Key.
  • Composite Key.
  • Sort or Control Key.

8. What is a Table/Relation?

A two-dimensional array of data that contains descriptive information about an entity is known as a table or relation. The relations are used to store information about an entity.

9. Differentiate between parent table and child table.

In a database relationship, the table which contains the foreign key is called the child table. While the table to which the foreign key refers is called the parent table.

For example, a table called Marks has foreign key Student ID which refers to the table Students. So the child table is Marks, and the parent table is Students

10. How is a relation formed in a database?

The relations are formed according to the attributes of the entity about which information is to be stored. The attributes of the entity become the fields of the relation. The name of the entity becomes the name of the relation. For example, Student is an entity so we can create a relation Students with fields like Roll Number, Name, Address etc.

11. Differentiate between Primary key, Candidate key and Alternate key.

A primary key is an attribute or a combination of attributes which can uniquely identify the rows in a table.

If a table has more than one such combination of keys, each combination is called a Candidate key.

The candidate keys which are not used as the primary key are called Alternate keys.

12. Differentiate between Primary key, Secondary key and Foreign key.

A primary key is an attribute or a combination of attributes which can uniquely identify the rows in a table.

A secondary key is an attribute or a combination of attributes for which there may exist more than one row in a table. So we can say that a secondary key non-uniquely identifies the rows in a table.

A foreign key in a table has a value which corresponds to the primary key of another table.

13. What is the purpose of an index file?

An index is a data structure that is used by DBMS to locate a particular record in a file more quickly. Indexes are used to speed up the sorting and searching process. Some indexes are created automatically in the related tables when relationships are defined. The performance of database is improved with these indexes.

14. Name 4 major components of a database system.

The four major components of a database system are:
  1. Data
  2. Hardware
  3. Software
  4. People (Users)

15. What are the properties of a relation?

The two important properties of a relation are:
  • No duplicate row exists.
  • Order of rows is insignificant.

16. What is an entity?

An entity is something or someone about which information is stored in a database. The entity must have a unique identifier, which is composed of one or more attributes.

17. Define attribute.

The named columns of the relation are called attributes. In a relation, each attribute has a unique name. For example, Roll Number, Student Name, Address.

18. Who is end user?

A user or an end user is a person who interacts with DBMS to perform different operations on database such as retrieving, updating, inserting and deleting data.

19. Who is a database administrator? What are his responsibilities?

A database administrator is responsible for the design, implementation, operation, management and maintenance of a database.

The major responsibilities include assigning permissions to different users, monitoring database usage, and solving any issues that may occur.

20. What is a View? What is its purpose?

Presentation of data for users is known as data view. By using this feature of DBMS, data from several relations can be retrieved and presented to the user as one table in different styles and formats. A view is also known as a virtual table.

The basic purpose of using a view is to keep data safe and secure from unauthorized users, because a view can only be used to view data not edit it. A view provides more flexibility and security in displaying data.
