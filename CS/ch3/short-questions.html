1. Define relationship with an example.

A logical connection between different entities is called a relationship.

For example, a relationship exists between a Student and Teacher because Teacher teaches Student.

2. What is one-to-one relationship? Give an example.

One-to-one cardinality means that one instance of entity A is related to only one instance of entity B. For example, Chairman runs a Department. The department can have only one chairman, and a chairman can be the chairman of only one department.

3. What is cardinality of a relation?

The number of instances of one entity that can be associated with each instance of another related entity is known as cardinality of relationship. The cardinality is expressed as one or many. For example, a Country can have only one Capital, and a Mother can have many Children.

4. List different types of relationships.

The different types of relationship are:
  • One-to-one.
  • One-to-many.
  • Many-to-many.
  • Recursive.

5. State the purpose of database design process.

The main objective of database design is to map the conceptual data model to an implementation model. A particular DBMS must be able to process this model.

6. What are the objectives of physical database design?

Physical design is the last stage of database design process. The main objective is to implement the database as a set of records, files, indexes and other data structures.

7. What is the logical database design?

The logical or conceptual model describes the data stored in the database. It contains the complete definition and details of the data to be stored in database. It also contains the rules and information about the structure and type of data.

8. What is data modeling?

When we want to prepare a database system the important thing is data. That data is required to fulfill the requirements. Data modeling is a way to represent required data. So the identification or representation of various data objects, events and the association or relationship between them is called data modeling. There are different data models such as ER Diagram.

9. Define modality.

Modality describes relationships as either mandatory or optional. It specifies absolute minimum number of relationships. The relationship is called optional if minimum number of required relations are zero. And it is mandatory if the minimum number is one.

10. What is meant by the term "Analysis"?

A process of studying the existing system is known as analysis. The basic purpose of analysis is to know which activities are performed in the current system. The analysis also determines what should take place in the new system.

11. What are the activities involved in data analysis?

Data analysis involves the activities of making Data Flow Diagrams (DFDs), Decision Tables and Decision Trees.

12. What are the ingredients of data modeling?

A data model consists of different elements or objects that are used to represent the data items. These objects are known as ingredients. Some of them are:
  1. Entities
  2. Attributes
  3. Relationships

13. What is the concept of data distribution strategy?

Many organizations today have distributed computing networks. For these organizations, a significant problem is physical database design, which is to decide at which locations (sites) in the network to store the data.

14. What is an ER diagram? Why is it used?

An Entity Relationship Diagram (ERD) is a symbolic representation of entities and their relationships in a database. ER diagrams use different symbols to represent different information of a database system.

ER diagrams are used to analyze existing databases to find and resolve problems in logic or deployment. ER diagrams make it easy to design and find problems in the design of databases.

15. What is project planning?

After requirements analysis, a proper planning and time schedule must be developed for different activities that are required to design a database system. In addition to a time schedule (deadlines) for different activities, the cost factors are also taken into consideration.

16. What is the purpose of feasibility study?

Feasibility study is conducted to investigate required database system. It determines whether the proposed system is affordable, possible, and acceptable. It also involves the identification and selection of area.
